This project is a fully functional web-based Support Ticket Management System built using PHP, MySQL, HTML, CSS, and JavaScript, designed to help any organisation, FiveM server, Discord community, or business efficiently manage user issues in a structured and trackable way. The system works by allowing users to log in, access a personalised dashboard, and create support tickets whenever they face a problem, want to request something, or report an issue. When a user submits a ticket, the system stores the ticket’s title, category, description, date, status, and user ID inside the MySQL database, marking it initially as Open. The backend uses PHP to validate inputs, manage sessions, process form submissions, and communicate with the database to save and retrieve ticket information. Once submitted, staff members and admins can log in to their own dashboard where they can view all user tickets, filter them by status (open, pending, resolved), open any ticket to read details, and send replies to communicate directly with the user. Each reply is stored in a “ticket conversation” table, allowing both user and staff to communicate in a chat-style thread inside the view_ticket.php page. The system also updates ticket status as the conversation progresses — for example, staff can set a ticket to In-Progress, and once the issue is solved, they can mark it as Closed, making the workflow easy to track. The interface is divided into modular files such as dashboard.php, ticket_list.php, create_ticket.php, view_ticket.php, sidebar.php, and topbar.php, all of which work together to display a clean, consistent user experience. The roles in the system include Users, who can only create tickets and view their own; Staff, who can view, reply, and close any ticket; and Admins, who have full control including managing all users and overseeing the entire ticket system. The system’s working process follows a clear and efficient cycle: a user logs in → creates a ticket → the system stores the ticket → staff receives the ticket → staff replies from the admin panel → user sees the reply → both communicate until the issue is fixed → staff marks it as closed → the ticket is archived for record-keeping. This creates a streamlined support workflow that ensures every issue is documented, traceable, and resolved professionally. The overall system is lightweight, modular, and easily extendable, making it suitable for any environment where organised, efficient ticket management is needed.
